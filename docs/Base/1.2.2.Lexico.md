# 1.2.2. Léxico (LAL — Léxico Ampliado da Linguagem)

## Objetivo do Documento

Este documento define o Léxico Ampliado da Linguagem (LAL) do projeto MonitoreSeuTreino, padronizando os principais termos do domínio para reduzir ambiguidades.

## Estrutura do Léxico

Cada termo é descrito com:

- **Identificador:** Lxx
- **Nome do termo**
- **Classificação:** Objeto, Ação (Verbo) ou Estado
- **Sinônimos:** termos equivalentes usados informalmente
- **Noção:** definição do termo no contexto do sistema
- **Impacto:** como o termo influencia o sistema (cadastros, regras, telas, dados, restrições)

## Termos do Léxico

### L01 — Usuário

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Praticante, Aluno, Pessoa usuária |
| Noção | Pessoa que utiliza o sistema para planejar e registrar sua rotina de treino semanal. |
| Impacto | Todos os dados (rotinas, sessões, histórico e resumos) devem estar associados a um Usuário. O sistema deve permitir identificar o Usuário para personalizar o Resumo Semanal e o Histórico. |

### L02 — Autenticação

| Campo | Descrição |
| ----- | --------- |
| Classificação | Ação (Verbo) |
| Sinônimos | Login, Entrar, Acessar conta |
| Noção | Processo de permitir que o Usuário acesse o sistema por meio de credenciais (ou mecanismo simplificado no MVP). |
| Impacto | Define quem pode ver/editar dados. |

### L03 — Ficha de Treino Semanal (Rotina)

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Rotina, Plano semanal, Programação semanal |
| Noção | Estrutura planejada pelo Usuário para organizar seus treinos na semana (ex.: Treino A/B/C, ou por dia da semana), contendo exercícios e parâmetros-alvo. |
| Impacto | É criada/alterada na funcionalidade "Criar/editar ficha". Pode servir como referência para gerar uma Sessão (execução real) sem precisar digitar tudo do zero. |

### L04 — Exercício

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Movimento, Atividade |
| Noção | Item que compõe a Ficha/Sessão, representando um movimento físico (ex.: agachamento, supino). |
| Impacto | Deve existir no catálogo/lista para o Usuário selecionar ao montar a Rotina. Estrutura base para organizar Séries, Repetições e Carga. |

### L05 — Parâmetros Planejados

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Meta do treino, Prescrição, Alvo |
| Noção | Conjunto de valores definidos na Rotina para orientar a execução (ex.: séries alvo, repetições alvo, observações). |
| Impacto | Usado para pré-preencher a Sessão ao registrar o treino. Ajuda a comparar "planejado vs realizado" (mesmo que o MVP não exiba comparação avançada). |

### L06 — Sessão de Treino (Execução)

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Sessão, Treino realizado, Registro do treino |
| Noção | Registro do treino executado em uma data/hora, com os valores realmente realizados (carga, reps, séries, observações). |
| Impacto | É o núcleo do Histórico e do Resumo Semanal. Deve permitir edição/correção conforme o requisito do MVP. |

### L07 — Registrar Sessão

| Campo | Descrição |
| ----- | --------- |
| Classificação | Ação (Verbo) |
| Sinônimos | Registrar treino, Salvar treino, Lançar treino |
| Noção | Ação pela qual o Usuário cria e preenche uma Sessão com os dados realizados. |
| Impacto | Dispara validações (campos obrigatórios, números não negativos). Ao concluir o registro, a sessão passa a compor o Histórico e o Resumo Semanal. |

### L08 — Série

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Set |
| Noção | Conjunto de repetições de um Exercício executadas continuamente antes de descansar. |
| Impacto | Compõe os dados da Sessão; cada exercício pode ter 1..N séries. Estrutura usada para registrar Repetições e Carga. |

### L09 — Repetições

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Reps |
| Noção | Quantidade de execuções completas de um Exercício dentro de uma Série. |
| Impacto | Usada para avaliar o volume realizado e alimentar o Histórico. Deve ser validada (ex.: inteiro não negativo) conforme regra do sistema. |

### L10 — Carga

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Peso, Quilos |
| Noção | Valor numérico representando o peso/resistência usada em um Exercício em determinada Série. |
| Impacto | Campo essencial na Sessão para exercícios de resistência. Deve ser validada (não negativa); pode aceitar decimais conforme decisão do time. |

### L11 — Observação

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Nota, Comentário |
| Noção | Texto livre associado a uma Sessão ou a um Exercício, para registrar detalhes como sensação, ajuste de técnica ou motivo de alteração. |
| Impacto | Ajuda na correção futura e na compreensão do Histórico. Não deve ser campo obrigatório. |

### L12 — Histórico de Treinos

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Histórico, Diário de treino, Registro cronológico |
| Noção | Conjunto de Sessões registradas por um Usuário, normalmente exibidas em ordem temporal. |
| Impacto | Deve permitir consulta e detalhamento de sessões. Serve como base para o Resumo Semanal. |

### L13 — Resumo Semanal (Monitoramento)

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Painel semanal, Dashboard semanal |
| Noção | Consolidação de informações da semana (ex.: quantidade de sessões registradas; opcionalmente volume/tempo). |
| Impacto | Deve usar uma regra de período (por padrão, segunda a domingo, se adotado). É um requisito mínimo do MVP para dar visibilidade à consistência. |

### L14 — Semana Selecionada

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | — |
| Noção | Intervalo de datas usado para calcular o Resumo Semanal e filtrar sessões no Histórico. |
| Impacto | Define como o sistema agrupa e exibe treinos. |

### L15 — Edição/Correção de Registro

| Campo | Descrição |
| ----- | --------- |
| Classificação | Ação (Verbo) |
| Sinônimos | Editar sessão, Corrigir treino, Ajustar valores |
| Noção | Ação que permite ao Usuário alterar dados de uma Sessão já registrada, para corrigir erros de digitação ou registro. |
| Impacto | Deve atualizar automaticamente o Histórico e o Resumo Semanal. Exige trilha de validação (ex.: impedir valores inválidos). (Opcional futuro) pode exigir auditoria/versionamento, mas não é obrigatório no MVP. |

### L16 — Sessão Concluída

| Campo | Descrição |
| ----- | --------- |
| Classificação | Estado |
| Sinônimos | Finalizada, Salva |
| Noção | Estado de uma Sessão após o registro ter sido salvo com sucesso. |
| Impacto | Sessões concluídas entram no Histórico e contam no Resumo Semanal. Pode ser reaberta para Edição/Correção, conforme regra do MVP. |

### L17 — Sessão em Rascunho

| Campo | Descrição |
| ----- | --------- |
| Classificação | Estado |
| Sinônimos | Não finalizada, Em edição |
| Noção | Sessão iniciada pelo Usuário, porém ainda não salva como concluída (ou não validada). |
| Impacto | Permite ao Usuário interromper e continuar depois (opcional no MVP). Se adotado, não deve contar no Resumo Semanal até ser concluída. |

### L18 — Validação de Dados

| Campo | Descrição |
| ----- | --------- |
| Classificação | Ação (Verbo) |
| Sinônimos | Validar campos, Regras de preenchimento |
| Noção | Conjunto de verificações aplicadas ao preencher Rotina e Sessão (ex.: obrigatoriedade, formato, faixa de valores). |
| Impacto | Reduz inconsistências (ex.: carga negativa, reps vazias). Deve ser aplicada tanto na UI quanto na persistência (se houver backend). |

### L19 — Persistência

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto |
| Sinônimos | Armazenamento, Banco de dados |
| Noção | Mecanismo pelo qual o sistema salva Rotinas e Sessões para que possam ser consultadas posteriormente. |
| Impacto | No MVP pode ser via armazenamento em nuvem (backend) e/ou armazenamento local do navegador. Impacta diretamente o requisito de Histórico e Resumo Semanal. |

### L20 — Aplicação Responsiva

| Campo | Descrição |
| ----- | --------- |
| Classificação | Objeto (característica do sistema) |
| Sinônimos | Responsivo, Mobile-friendly |
| Noção | Propriedade da aplicação Web de se adaptar a diferentes tamanhos de tela (desktop e celular). |
| Impacto | Requisito implícito do projeto, pois o uso em academia pode ocorrer no celular via navegador. Influencia decisões de layout, componentes e testes de interface. |
